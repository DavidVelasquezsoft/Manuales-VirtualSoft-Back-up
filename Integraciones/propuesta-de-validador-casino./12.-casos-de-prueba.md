# 12. Casos de prueba

Los casos de prueba a modelar se encontrarán en la carpeta de EGT, en donde se\
debe tener en cuenta “abrir juego, autenticación, balance, debito, crédito, rollback,\
rondas gratis.

<figure><img src="../.gitbook/assets/image (12).png" alt=""><figcaption></figcaption></figure>

#### 12.1. Iniciar juego

En esta colección encontrará diferentes métodos con el fin de logear al usuario en la\
plataforma y pueda realizar transacciones, estos métodos ya están predefinidos y no se\
deben modificar a menos que se requiera hacer las pruebas con un usuario diferente.

<figure><img src="../.gitbook/assets/image (13).png" alt=""><figcaption></figcaption></figure>

En “Iniciar Juego” encontrará las variables que permiten iniciar el juego, como URLs, id&#x20;usuario, monto apostar, monto ganado.

#### 12.2. Abrir juego

El método de abrir juego es igual para todos, simplemente se debe modificar el gameId\
y Provider, ya que para cada juego es diferente.

<figure><img src="../.gitbook/assets/image (2).png" alt=""><figcaption></figcaption></figure>

#### 12.3. Autenticación

En el método de autenticación se debe configurar el body, y la validación de los datos\
recibidos en el script, en el archivo de “Auth fallido” se deben configurar los casos de\
prueba que hacen que la autenticación falle por ejemplo token invalido o usuario no\
existe.

<figure><img src="../.gitbook/assets/image (3).png" alt=""><figcaption></figcaption></figure>

#### 12.4.  Balance

En el método para obtener el saldo de la billetera también se debe modificar el body a\
enviar y los datos que se reciben mediante el script. En la colección de “balance fallido”\
también se deben configurar los casos de prueba que hacen que la transacción sea\
fallida, como usuario inválido o transacción fallida.

<figure><img src="../.gitbook/assets/image (4).png" alt=""><figcaption></figcaption></figure>

#### 12.5.  Debito

En la colección de débito se deben agregar los siguientes casos de prueba:\
• Debito exitoso.\
• Debito + Debito\
• Debito condiciones\
Igualmente se debe configurar el body a enviar y los datos recibidos.

<figure><img src="../.gitbook/assets/image (5).png" alt=""><figcaption></figcaption></figure>

#### 12.6. Crédito

En la colección de crédito se deben agregar los siguiente casos de prueba:\
• Debito + crédito\
• Debito + credito – cuando la transacción no existe\
• Debito + credito + credito – cuando la transacción ya fue procesada\
• Auth + balance + debito + credito\
• Debito + credito + debito + credito\
• Debito (ronda abierta) + Credito (ronda abierta) + credito (ronda cerrada) +\
credito\
Igualmente se debe configurar el body a enviar y los datos recibidos.

<figure><img src="../.gitbook/assets/image (6).png" alt=""><figcaption></figcaption></figure>

#### 12.7. Rollback

En la colección de rollback se deben agregar los siguientes casos de prueba:\
• Debito + rollback\
• Debito + rollback + credito\
• Debito + rollback + rollback\
• Debito + Credito + Rollback\
• Auth + Balance + Debito + Rollback\
Igualmente se debe configurar el body a enviar y los datos recibidos.

<figure><img src="../.gitbook/assets/image (7).png" alt=""><figcaption></figcaption></figure>

#### 12.8. Rondas gratis

Si aplica en el juego también se deben agregar los casos de prueba para rondas gratis.
