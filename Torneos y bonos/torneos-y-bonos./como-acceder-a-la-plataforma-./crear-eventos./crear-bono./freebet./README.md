#### **4.3. Moneda**

<details>

<summary>🔽<strong>Parámetros de configuración por moneda</strong></summary>

Al dar clic en la moneda correspondiente al país con el que se ingresó a la plataforma, se desplegarán las siguientes configuraciones:

<figure><img src="https://1957026231-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FrLdGx9JdTz3uLoquKvJw%2Fuploads%2FKJUaOkTbOWgYVVTfeYn4%2Fimage.png?alt=media&#x26;token=09a7ebd2-5fce-4d3e-b545-b49d1ab02d23" alt=""><figcaption><p>Figura #2: Captura de pantalla configuración moneda.</p></figcaption></figure>

El bono funciona bajo una lógica de cobertura por apuesta, definida por los valores mínimo y máximo configurados.

**Comportamiento del bono:**

* Si la apuesta es **menor al Mínimo valor fijo**, el sistema no utiliza el bono y el valor se descuenta del **saldo apuestas** del usuario.
* Si la apuesta es **igual o superior al Mínimo valor fijo**, el sistema utilizará el bono hasta el límite permitido.
* Si la apuesta supera el **Máximo valor fijo**, el sistema aplicará únicamente el valor máximo configurado del bono y el excedente se descontará del saldo apuestas del usuario.

De esta manera, el bono actúa como un respaldo parcial o total dependiendo del monto apostado y los límites configurados.

### 5. Reglas y validaciones.

* La configuración del campo **`Cliente puede recibir otros bonos adicionales después del actual?`** solo aplica mientras el bono está activ&#x6F;**.**
* El bono se asignará únicamente a los usuarios que cumplan los criterios definidos _(tipo, campaña, cantidad de jugadores, códigos o carga de archivo CSV)_.
* Cuando un bono **Freebet** esté asignado y activo, el usuario visualizará un apartado de **Saldo Freebet** en la plataforma; al realizar una apuesta que cumpla las condiciones configuradas, el valor correspondiente se descontará de este saldo, consumiendo el bono según el monto utilizado en dicha apuesta.
* En la plataforma Usuarios Online siempre se visualizará el saldo mínimo configurado.
